Humansense Android App

Experimental Features

By default, the application will log your GPS, GSM, and Wifi information. It will start automatically when the phone boots up, and can be controlled via the HumanSense application. To provide location labels to assist in ongoing research into localization, please use the Location Labeler widget. This can be placed onto the home screen of the phone by long-clicking on a blank region, then selecting "Add Widget", then adding the "Location Labeler" widget. Whenever you are stationary in a location for at least one minute, you can click on "Where are you now?", and provide a label.

To see the app do something interesting, I suggest trying out either the activity recognition plugin or the location recognition plugin.

Activity Recognition

This plugin allows you to build models for physical activities (e.g., running, walking, hopping on one foot, etc.), and then later recognize which activity you are peforming. It can also be used to build models for various people's gaits (i.e., style of walking), and then it can detect who is carrying the phone.

To try out this plugin, you need to enable the sensor input plugin and the classifier output plugin. To enable the sensor input plugin, go to the Settings, then to Input Plugins, then click the check box next to Sensor Logging Plugin. In the Sensor Speed setting, choose Fastest. Then go back into the Settings, choose Output Plugins, and enable the Simple Classifier Plugin. The classifier is only active when the phone is in motion, and the Lingering parameters adjust how sensitive the classifier is. I've found that setting the Lingering Power Threshold to 100, and the Lingering Power Window to 25 work well. The Manage Model Files setting allows you to choose which models the classifier uses, and we'll get to that in a moment.

Now, return to the main Humansense application screen, and click Start Service. Two new buttons should appear at the bottom of the screen. Build is used to build new models (i.e., create new activities or people to be recognized). Clicking Build will cause the device to begin collecting samples from the accelerometers. To create a new activity model, click Build, put the phone in your pocket (locking the screen first so that no buttons are pressed accidentally while in your pocket), perform the activity for a short period of time (10 seconds or so), remove the phone from your pocket (unlock it), and click Stop Building. A new dialog will appear, showing you the collected accelerometer data. The purpose of this dialog is to trim off the initial and final few seconds, which contain noisy data from when you put the phone in your pocket and removed it. Select a nice section of the data by clicking and dragging on the data. A new dialog will appear allowing you to assign a name to this data. If you accidentally clicked on a bad portion of the data, click Cancel to return to the previous screen to re-select a segment. Once you have selected a good segment and assigned it a name that you will remember, click OK. This will return you to the main application screen. Repeat the process to create a few activities, or pass the phone to your friends and collect data from them walking around with the phone in their pocket.

Once you have collected a few activities, the next step is to perform one of the activities again, and have the phone guess which one you are performing. The first step is to select a set of candidate activities from which the classifier will choose. To do this, go into the Settings, into Output Plugins, and then select Manage Model Files. Here you will see a list of all of the models that you have created. Check off the models that you want to use for this session. At the moment, the application starts to stop performing well when more than 10 or so models are active. Once you have selected the models you would like to consider, click Save to save your model selection. Now go back to your phone's home screen (not the application home screen), and add the widget in which the results will be displayed. To do this, long-click on an empty portion of the home screen, select Widget and add the Lingering Notification Widget. The widget should appear on the home screen.

Now, to start classifying, go back into the Humansense application, ensure that the service has been started, and then click the button on the bottom of the screen that says Classify. Now, if you go back to your home scren and look at the widget, you should notice that some text has appeared. The first two lines simply count the number of samples that are considered to be lingering (the phone is not moving), or otherwise. Below these two lines another set of lines should appear. There will be one line per activity, labeled according to the name that you chose when the model was built. Next to these activities will be the probability that the classifier has assigned to each activity. If you lock the phone and put it in your pocket, then perform one of the activities for a few seconds, when you unlock the phone the value next to the activity that you performed should be larger than the value for the other activities. As this is a work in progress, and part of an active research program, please let me know how it works.

Location Recognition

This plugin will learn to recognize places that you spend time, based on building fingerprints from observable wifi base stations. To try it out, first install the application, then start the app and go into the Settings menu. Choose Input Plugins, and enable the Wifi Logging Plugin, and in the Wifi Logging Interval setting, select 1 minute. Next, go back into the Settings menu and choose the Output Plugins settings. Enable the Location Clustering Plugin. Now, go back to your home screen and press and hold on an empty part the screen. This should bring up a menu that allows you to add a Widget to your home screen. Add the Location Status widget. Now, go back into the Humansense application and start the service.

The service collects data in the background, so you can go back to your home screen (press the Home button) and use other apps. The widget on your home screen should be updated with your status (green means you're moving, read means you're stationary) and a location ID, which is UNKNOWN if you have not been stationary in a location for long enough for it to be considered a significant location, or a positive integer denoting the ID associated with the current location. If a positive integer appears, you can assign a textual label to this location by clicking on the widget. This should bring up a new dialog window that allows you to provide a name for the location. From then on, if you are in that location, the name should appear instead of the ID number.

This is a work in progress, so please let me know how it works.
